# 合力牧业农牧乳一体化经营管理平台使用说明

## 1. 推荐演示路径

进入系统后先看“首页驾驶舱”。首页现在只聚焦四件事：

- 今天生产怎么样
- 今天有没有异常
- 今天谁需要处理任务
- 四大业务板块运行是否正常

然后根据演示对象进入对应工作台：

- 合力牧业奶牛场工作台
- 欧力菲德肉牛场工作台
- 欧力菲德饲料厂工作台
- 欧力菲德乳品厂工作台

具体台账明细从工作台卡片进入，不建议一开始直接讲所有明细页。

## 2. 三大工作模式

左侧菜单上方可以切换工作模式：

- 经营总览：适合老板、集团管理层、场长、厂长查看整体经营。
- 业务操作：适合员工、司机、仓库、质检、生产人员处理今日任务。
- 平台配置：适合系统管理员、演示人员、开发人员展示平台能力。

切换工作模式不会丢失数据，只是前端菜单展示发生变化。

## 3. 角色视图

顶部可以切换角色视图。角色切换后会影响菜单范围和默认入口：

- 集团管理员：默认首页驾驶舱
- 奶牛场场长：默认合力牧业奶牛场工作台
- 肉牛场场长：默认欧力菲德肉牛场工作台
- 饲料厂厂长：默认欧力菲德饲料厂工作台
- 乳品厂厂长：默认欧力菲德乳品厂工作台
- 物流主管：默认配送任务
- 财务人员：默认经营报表
- 普通员工：默认员工工作台
- 司机：默认配送任务
- 只读访客：默认平台能力总览

当前仍然是前端模拟权限，不是真实登录鉴权。

## 4. 四大业务工作台

四个工作台是业务讲解的主入口：

- 奶牛场工作台：产奶、牛舍、繁育、防疫、饲喂、原奶配送、库存预警。
- 肉牛场工作台：存栏、育肥批次、称重增重、可出栏、生物资产。
- 饲料厂工作台：原料、成品、生产、调拨、采购、供应商、过磅。
- 乳品厂工作台：原奶接收、质检、生产计划、灌装、成品库存、销售订单。

每个工作台上方看指标，中间看趋势或状态，下方进入具体明细页。

## 5. 平台能力总览

“平台能力总览”适合给企业负责人或厂商讲解系统能力，包括：

- 四大业务板块
- 业务流、物流、资金流、数据流、权限流、消息流、设备流
- 工单、消息、饲喂、牛只事件、追溯、进销存、账本、报表、接口、AI、员工端
- 后续后端数据库、小程序、企业微信、IoT 和 AI 接入路线

## 6. 二级明细页

BusinessSection 仍然保留，用于具体台账明细，例如奶牛档案、饲料生产、销售订单、成品库存等。

这些页面不再全部暴露为一级菜单，而是从工作台、快捷入口或二级明细入口进入。

## 7. 移动端员工工作台

员工工作台模拟未来小程序 / H5 员工端，适合手机打开，包含：

- 今日任务
- 工单处理
- 消息提醒
- 产奶录入
- 饲喂执行
- 异常上报
- 扫一扫占位
- 配送确认

## 8. 数据说明

当前为纯前端 localStorage 演示版，不接真实后端、数据库、IoT、AI API、小程序或企业微信。

数据存储在浏览器：

```text
heli-smart-pasture-platform-v1
```

系统设置页可以导出 JSON、导入 JSON、重置示例数据和清空本机数据。

## 9. 本地运行

```powershell
cd E:\Projects\JAVA\heli-smart-pasture-demo
npm install
npm run dev -- --host 0.0.0.0
```

电脑访问：

```text
http://127.0.0.1:5173
```

手机与电脑连接同一 Wi-Fi 后，在 Safari 输入：

```text
http://电脑IP:5173
```

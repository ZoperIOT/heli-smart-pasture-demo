# 合力牧业农牧乳一体化经营管理平台使用说明

## 1. 推荐演示路径

进入系统后先看“首页驾驶舱”。首页适合先讲清楚四件事：

- 今天四大板块运行是否正常。
- 今天有哪些待办工单和异常提醒。
- 生产、配送、库存、销售和资金有什么联动。
- 老板、场长、厂长、财务和员工分别从哪里进入自己的工作台。

然后根据演示对象进入对应工作台：

- 合力牧业奶牛场工作台
- 欧力菲德肉牛场工作台
- 欧力菲德饲料厂工作台
- 欧力菲德乳品厂工作台

具体台账明细从工作台卡片、快捷操作或二级明细入口进入，不建议一开始直接逐个讲所有明细页。

## 2. 六个菜单模式

左侧菜单上方可以切换菜单模式：

- 经营总览：适合老板、集团管理层、场长和厂长查看整体经营。
- 业务流：适合员工、司机、仓库、质检、生产人员处理具体业务动作。
- 资金流：适合财务人员查看账本、成本、利润、应收应付和报表。
- 数据流：适合管理层、信息化负责人和厂商查看追溯、分析、接口、日志和未来 IoT 示例。
- 消息流：适合查看消息提醒、工单提醒、审批提醒和移动工作台。
- 权限与配置：适合系统管理员、演示人员和开发人员展示角色、规则、知识库和系统设置。

系统强调“五流合一”：业务流、资金流、数据流、消息流、权限流。配送、车辆和过磅归入业务流；设备状态和环境监控作为未来 IoT 接入能力示例放在数据流中。

## 3. 角色视图

顶部可以切换角色视图，并会同时显示当前组织。角色切换后会影响默认入口和可见菜单：

- 集团管理员：默认首页驾驶舱。
- 奶牛场场长：默认合力牧业奶牛场工作台。
- 肉牛场场长：默认欧力菲德肉牛场工作台。
- 饲料厂厂长：默认欧力菲德饲料厂工作台。
- 乳品厂厂长：默认欧力菲德乳品厂工作台。
- 物流主管：默认配送任务。
- 财务人员：默认经营报表。
- 普通员工：默认员工工作台。
- 司机：默认配送任务。
- 只读访客：默认平台能力总览，并隐藏新增、编辑、删除类入口。

当前仍然是前端模拟权限，不是真实登录鉴权。

## 4. 四大业务工作台

四个工作台是业务讲解的主入口。每个工作台都围绕“今日待办、快捷操作、进行中工单、异常提醒、历史工单、关键指标、最近业务动态”组织信息。

- 合力牧业奶牛场工作台：奶牛档案、牛舍、分群、饲喂、繁育、防疫、产奶、原奶配送和库存预警。
- 欧力菲德肉牛场工作台：肉牛档案、育肥批次、称重增重、出栏销售、生物资产和库存。
- 欧力菲德饲料厂工作台：原料、成品、生产、内部调拨、采购、供应商和过磅。
- 欧力菲德乳品厂工作台：原奶接收、质检、生产计划、灌装、成品库存、销售订单和退货。

## 5. 重点演示页面

- 首页驾驶舱：讲老板一眼看经营。
- 四大业务工作台：讲不同组织如何处理今天的工作。
- 工单中心：讲每日任务如何统一派发、跟踪和闭环。
- 经营报表中心：讲收入、支出、利润、库存、生产、异常和应收应付如何汇总。
- 数据大屏：讲集团化指标展示。
- 平台能力总览：讲五流合一、业务闭环和后续接口规划。
- AI 经营助手：讲自然语言查询经营情况的价值。

## 6. 二级明细页

二级明细页用于具体台账，例如奶牛档案、饲料生产、原奶接收、成品库存、销售订单、地磅过磅、客户供应商、员工岗位等。

这些页面仍然保留，方便后续接真实后端时逐步替换 mock 数据。

## 7. 移动端员工工作台

员工工作台模拟未来 H5 / 小程序员工端，适合手机打开，包含：

- 今日任务
- 工单处理
- 消息提醒
- 产奶录入
- 饲喂执行
- 异常上报
- 扫一扫占位
- 配送确认

当前只是前端页面，不接真实小程序、摄像头、定位或扫码接口。

## 8. 数据说明

当前为纯前端 localStorage 演示版，不接真实后端、数据库、IoT、AI API、小程序或企业微信。

数据存储在浏览器：

```text
heli-smart-pasture-platform-v1
```

系统设置页可以导出 JSON、导入 JSON、重置示例数据和清空本机数据。

## 9. 本地运行

```powershell
cd E:\Projects\JAVA\heli-smart-pasture-demo
npm install
npm run dev -- --host 0.0.0.0
```

电脑访问：

```text
http://127.0.0.1:5173
```

手机与电脑连接同一 Wi-Fi 后，在 Safari 输入：

```text
http://电脑IP:5173
```

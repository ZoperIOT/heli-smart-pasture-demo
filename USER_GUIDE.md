# 合力牧业员工端作业系统使用说明

## 1. 推荐演示路径

打开系统后先看“今日工作台”。第一屏应先讲清楚：

- 今天有哪些待办任务。
- 哪些任务比较紧急。
- 我有哪些工单要处理。
- 我可以快速提交哪些现场记录。
- 最近有没有新消息。

然后按一线员工真实工作顺序演示：

1. 进入“饲喂管理”，选择今日饲喂任务，填写实际投料、剩料和采食情况。
2. 进入“产奶管理”，提交早班或晚班产奶记录，选择是否送检和生成配送任务。
3. 进入“繁育管理”，录入发情观察、配种、妊检或产犊记录。
4. 进入“牛只管理”，搜索牛号，查看档案并上报疾病、用药、转群等事件。
5. 进入“库存管理”，提交领料申请、入库、出库或盘点记录。
6. 进入“质检管理”，处理原奶、饲料或成品待检任务。
7. 进入“异常上报”，提交现场异常，系统自动生成工单和消息。
8. 进入“工单”，处理待处理、处理中、待复核和已完成状态。
9. 进入“我的记录”，查看当前用户提交和处理过的业务记录。

## 2. 底部导航

底部只保留五个移动端入口：

- 首页：今天要干什么。
- 作业：饲喂、产奶、繁育、牛只、库存、质检、异常。
- 工单：处理分配给我的工单，管理员可看全部工单。
- 消息：查看任务提醒、工单提醒、库存预警、质检提醒、繁育提醒。
- 我的：个人身份、我的统计、重置示例数据和复制本机 JSON。

## 3. 身份切换

顶部可以切换三个身份：

- 员工：主要用于提交业务记录和处理分配给自己的工单。
- 管理员：可以查看全部员工记录、全部工单和全部任务。
- 只读访客：只能查看示例数据，不能提交或处理。

演示只读访客时，点击提交或处理按钮会提示不能执行该操作。

## 4. 重点业务模块

### 饲喂管理

展示今日饲喂任务、饲喂执行表单和联动效果。字段包括牛舍、牛群、班次、配方、饲料批次、计划投喂量、实际投喂量、剩料量、饮水情况、采食情况、执行人、执行时间、是否自动扣减库存、异常说明和现场照片占位。

提交后会更新饲喂任务，写入我的记录和操作日志；异常投喂、剩料过高或采食异常会自动生成工单和消息；选择自动扣减库存会减少 mock 库存。

### 产奶管理

展示今日挤奶任务和产奶录入表单。字段包括班次、牛舍、牛群、泌乳牛头数、挤奶员、总产奶量、合格奶量、异常奶量、入罐编号、原奶温度、蛋白、脂肪、体细胞、菌落总数、是否送检和是否生成原奶配送任务。

提交后会更新挤奶任务，写入我的记录；选择送检会生成质检任务；异常奶或温度异常会生成质量工单和消息。

### 繁育管理

支持发情观察、配种记录、妊检记录和产犊记录。配种和妊检会生成后续提醒，记录也会写入牛只事件。

### 牛只管理

支持按牛号、牛舍、牛群和状态搜索。牛只卡片展示位置、状态、健康、最近饲喂、最近产奶、最近繁育和最近防疫。疾病、用药、死亡、异常行为等事件会生成工单。

### 库存管理

支持库存总览、领料申请、入库、出库和盘点。库存低于安全库存会生成预警消息，领料申请会生成库存工单。

### 质检管理

支持待检任务、原奶质检、饲料质检和成品质检。不合格会自动生成质量异常工单和消息。

### 异常上报

异常类型包括牛只异常、饲喂异常、产奶异常、质检异常、库存异常、安全隐患等。提交后自动生成工单、消息和我的记录。

## 5. 数据说明

当前为纯前端 localStorage 演示版，不接真实后端、数据库、企业微信、小程序、IoT 或 AI 服务。

数据存储在浏览器：

```text
heli-smart-pasture-platform-v1
```

个人中心可以重置示例数据，也可以复制本机 JSON 数据。

## 6. 本地运行

```powershell
cd E:\Projects\JAVA\heli-smart-pasture-demo
npm install
npm run dev -- --host 0.0.0.0
```

电脑访问：

```text
http://127.0.0.1:5173
```

手机与电脑连接同一 Wi-Fi 后，在 Safari 输入：

```text
http://电脑IP:5173
```

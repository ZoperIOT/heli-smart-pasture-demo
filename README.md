# 合力牧业员工端作业系统演示版

当前项目已调整为“合力牧业企业微信员工工作台 / 小程序员工端”风格的前端演示系统。它面向中小型畜牧企业的一线员工和管理员，重点服务每天的移动端作业、填报、工单和消息处理。

系统仍然是纯前端 localStorage 演示版，不接真实后端、真实数据库、真实企业微信、真实小程序、真实 IoT 设备或真实 AI API。

## 当前定位

- 产品名称：合力牧业员工端作业系统演示版
- 使用场景：手机 H5、企业微信工作台、小程序员工端原型演示
- 核心目标：今日智能工单、业务填报、异常上报、交接班、工单处理、消息提醒、草稿暂存、历史记录
- 默认入口：`/app`，根路径 `/` 也直接进入员工工作台

## 角色

系统只保留三个移动端身份：

- 员工：查看今日待办，提交饲喂、产奶、繁育、牛只、库存、质检和异常记录，处理分配给自己的工单。
- 管理员：查看全部任务、全部员工记录、全部工单，主动创建和派发工单，处理异常、审核库存申请，查看简单统计。
- 只读访客：只能查看示例数据，不能提交表单、处理工单或修改数据。

顶部可以直接切换身份。只读访客点击提交或处理类按钮时，会提示“当前为只读演示模式，不能执行该操作。”

## 主导航

移动端底部导航只有四个入口：

- 首页
- 工单
- 消息
- 我的

旧版偏展示、偏方案、偏复杂分析的入口，已从员工端主界面隐藏或重定向到员工工作台。

## 核心页面

- `/app`：员工端首页，展示今日待办、紧急任务、待处理工单、今日已提交、快捷操作、最近消息和我的记录。
- `/tasks`：今日智能工单和作业入口。
- `/work`：作业中心，等同于今日智能工单入口。
- `/feeding`：饲喂管理，包含饲喂任务、执行记录、偏差率、剩料率、干物质采食量、自动扣减库存和异常联动。
- `/milk`：产奶管理，包含挤奶任务、班次奶量、单牛平均产奶、异常奶比例、异常奶登记、入罐、原奶指标和送检联动。
- `/breeding`：繁育管理，包含发情观察、配种记录、妊检记录、产犊记录和繁育提醒。
- `/cattle`：牛只管理，包含快速查牛、牛只档案、事件上报、转群/疾病/用药/防疫等现场记录。
- `/inventory`：库存管理，包含库存总览、领料申请、入库、出库、盘点和低库存预警。
- `/quality`：质检管理，包含待检任务、原奶质检、饲料质检、成品质检和不合格联动。
- `/reports`、`/exceptions`：异常上报，提交后自动生成工单和消息。
- `/work-orders`：工单处理。员工处理自己的工单；管理员查看工单看板、派发工单、复核/驳回、取消工单和查看统计。
- `/handover`：交接班，记录未完成事项、异常牛只、异常库存、异常质检和接班确认。
- `/manuals`：操作手册，查看饲喂、挤奶、质检等现场规范。
- `/messages`：业务消息通知，和任务、工单、库存、质检、繁育提醒绑定。
- `/records`：我的记录，查看当前用户提交和处理过的业务记录。
- `/profile`：个人中心，切换身份、查看我的统计、重置示例数据、复制本机 JSON 数据。

## 业务联动

前端通过 localStorage 模拟以下闭环：

- 提交饲喂记录：更新饲喂任务状态，写入我的记录和操作日志；选择自动扣减库存时减少饲料库存；投喂超量、剩料过高或采食异常时生成工单和消息；库存不足时生成库存预警。
- 提交产奶记录：更新挤奶任务状态，写入我的记录和操作日志；选择送检时生成质检任务；选择生成配送任务时生成原奶配送任务；异常奶或温度异常时生成质量工单和消息。
- 提交繁育记录：写入繁育记录和牛只事件；配种、妊检等记录会生成后续提醒消息。
- 提交牛只事件：更新牛只状态；疾病、用药、死亡、异常行为等事件自动生成工单和消息。
- 提交库存申请：生成领料申请、库存工单和审批提醒消息；入库、出库、盘点会更新 mock 库存。
- 提交质检记录：更新待检任务；不合格时生成质量异常工单和消息。
- 提交交接班：写入交接班记录、我的记录和操作日志；有未完成或异常事项时生成待办、工单和消息。
- 提交异常上报：自动生成工单、消息、我的记录和操作日志。
- 管理员派发工单：写入 `workOrders` 和 `smartWorkOrders`，推送给被派发员工；员工提交结果后按“是否需要复核”进入待复核或已完成。
- 处理工单：状态变更后生成消息，并写入我的记录和操作日志。
- 保存草稿：表单可模拟离线暂存，草稿保存到 localStorage，并在我的记录和个人中心显示。

## 技术栈

- React
- Vite
- Tailwind CSS
- React Router
- lucide-react
- localStorage

项目仍保留 Recharts 依赖，但员工端主界面不再使用复杂图表。扫码、拍照、RFID、称重、TMR、地磅等硬件能力当前只保留占位入口或字段，不接真实硬件。

后续真实项目可以扩展为 PC 管理后台 + 企业微信员工端 + 小程序员工端 + 统一后端数据库。

## 本地运行

```powershell
cd E:\Projects\JAVA\heli-smart-pasture-demo
npm install
npm run dev -- --host 0.0.0.0
```

电脑访问：

```text
http://127.0.0.1:5173
```

```text
http://电脑IP:5173
```
